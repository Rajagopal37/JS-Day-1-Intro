Difference between HTTP1.1 vs HTTP2

HTTP 1.1 HTTP 2.0

Performance:

In HTTP/1.1, multiple requests from a client to a server typically require separate connections. This can lead to inefficiencies and increased latency, especially for websites with many resources to load.

HTTP/2 addresses this issue through features such as multiplexing, header compression, and server push. Multiplexing allows multiple requests and responses to be sent and received over a single connection simultaneously, reducing latency and improving overall performance.

Header Compression:

Headers in HTTP/1.1 are sent as plaintext and are not compressed. This can result in redundant data transmission, especially for large headers or repeated headers across multiple requests.

HTTP/2 uses header compression, which significantly reduces the size of headers sent over the wire. This helps in reducing overhead and improving performance, especially for requests with large or repeated headers.

Server Push:

In HTTP/1.1, servers can only respond to client requests. If a client needs additional resources after receiving the initial response, it must send additional requests to the server.

HTTP/2 introduces server push, a feature that allows servers to push additional resources to the client before the client requests them. This can help in reducing latency and improving performance by preemptively sending resources that the client is likely to request.

Binary Protocol:

HTTP/1.1 uses a textual format for its protocol messages, which can be inefficient to parse and process.

HTTP/2 uses a binary protocol, which is more efficient to parse and process, leading to better performance and reduced overhead.

Connection Multiplexing:

Each resource request typically requires a separate TCP connection, leading to the potential for head-of-line blocking issues.

HTTP/2 supports multiplexing, allowing multiple requests and responses to be interleaved and sent over a single connection. This improves efficiency and reduces latency by avoiding the overhead of establishing multiple connections.
